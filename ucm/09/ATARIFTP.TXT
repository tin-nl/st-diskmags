usually under a high load most of the time (many people accessing them
at the same time).
Mirror sites contain the exact same directories and files, but have
different entry points. Try to use the one closest to you geographically
and not during business hours (in that geographic area).

MAIL SERVERS
------------
Mail servers can be used to get hold of the same software as through FTP,
but instead of using a lot of download time online, files can be 'ordered'
via email.
In order to get information about how to do this, send an email to the
various addresses below with the word:  help
in the mail itself. If possible, do not include any 'signature' in the mail,
as the automated system which replies to these mails tries to interpret any
message (and with a certain amount of errors no response to your requests
will be given at all).

HOW TO GET FILES VIA FTP
------------------------
Below follows a step-by-step guide on how to get hold of files via FTP.
Your specific version of the FTP program might differ slightly (it might
have extra functions and be more user friendly etc.) But the main procedure
is likely to be more or less the same.

The examples below are taken from a Unix environment using the usual
'ftp' program. The more user friendly 'ncftp' is recommended though,
as it's easier and more efficient to use.


[1] Starting FTP/logging into FTP site
======================================

What to type                                           What it does
--------------------------------------------------------------------------
ftp                                                       [ENTER FTP MODE]
open FTPsite-address  (i.e: open ftp.funet.fi)         [OPEN THE FTP SITE]
anonymous             (when asked for username)          [INSERT USERNAME]
USERNAME@SITE (your email address- i.e. hallvart@sn.no)  [INSERT PASSWORD]

 Example:
---------------------------------------
  oslonett.no% ftp                                        [ENTER FTP MODE]
  ftp> open ftp.cs.tu-berlin.de                        [OPEN THE FTP SITE]
  Connected to caramba.cs.tu-berlin.de.
  220 caramba FTP server (wu-2.4/fb13[34] 18-Feb-1995) ready.
  Name (ftp.cs.tu-berlin.de:celine): anonymous           [INSERT USERNAME]
  331 Guest login ok, send your complete e-mail address as password.
  Password: celine@geonet.com               [INSERT PASSWORD- (NOT SHOWN)]
  230-You have reached Berlin, Germany.  Access is allowed all day.
---------------------------------------


* You can also enter a site and start ftp at the same time:

ftp FTPsite-address  (i.e: ftp ftp.cs.tu-berlin.de)  [START FTP/OPEN SITE]

 Example:
---------------------------------------
  oslonett.no% ftp ftp.cs.tu-berlin.de               [START FTP/OPEN SITE]
  Connected to caramba.cs.tu-berlin.de.
---------------------------------------

* It's also possible to use the IP number instead of the usual FTP
  site address:

ftp IP.number  (i.e: ftp 130.149.144.4)  [START FTP/OPEN SITE WITH IP NO.]

 Example:
---------------------------------------
  oslonett.no% ftp 130.149.144.4         [START FTP/OPEN SITE WITH IP NO.]
  Connected to 130.149.144.4.
  220 caramba FTP server (wu-2.4/fb13[34] 18-Feb-1995) ready.
---------------------------------------

[2] Preparing for file downloads
================================

cd /directory/   (i.e: cd/pub/atari/)          [MOVE TO DESIRED DIRECTORY]
bin                                          [SET TRANSFER MODE TO BINARY]
ls -l  (or: dir)             [SHOWS A DIRECTORY LISTING WITH FILE-LENGTHS]

 ** Some FTP sites don't accept 'ls -l' for some reason. In those rare cases
 use 'ls' instead (or use 'dir').

 Example:
---------------------------------------
  ftp> cd Music/                               [MOVE TO DESIRED DIRECTORY]
  ftp> bin                                   [SET TRANSFER MODE TO BINARY]
  200 Type set to I.
  ftp> ls -l                 [SHOWS A DIRECTORY LISTING WITH FILE-LENGTHS]
  150 Opening BINARY mode data connection for /bin/ls.
  total 4624
  -rw-rw-r--   1 ftpadm   atarimod    3409 Feb 19 12:48 00INDEX
  -rw-rw-r--   1 ftpadm   atarimod   37419 Mar 18  1993 50play12.lzh
  -rw-rw-r--   1 hesche   atarimod  105936 Jul 24  1995 525_v211.lzh
  -rw-rw-r--   1 ftpadm   atarimod  168363 Mar 10  1993 accomp25.lzh
---------------------------------------

* If you want to go back one directory:

          cd ..                                    [GO BACK ONE DIRECTORY]
          cd /.                       [GO DIRECTLY BACK TO ROOT DIRECTORY]

[3] Downloading files
=====================
get filename  (i.e: get letemfly.zip)                           [GET FILE]

 Example:
---------------------------------------
  ftp> get setseek.zip                                          [GET FILE]
  200 PORT command successful.
  150 Opening BINARY mode data connection for setseek.zip (3825 bytes).
  #######
  226 Transfer complete.
  ftp>
---------------------------------------

Several files at once:
----------------------
mget filenames (i.e: mget ao.lha xyz.zip ps.txt)       [GET SEVERAL FILES]

 Example:
---------------------------------------
  ftp> mget silkmous.zip snip.zip      [GET 'SILKMOUS.ZIP' AND 'SNIP.ZIP']
  mget silkmous.zip? y                      [DO YOU WANT THIS FILE (y/n)?]
  150 Opening BINARY mode data connection for silkmous.zip (22086 bytes).
  ############################################
  226 Transfer complete.
  mget snip.zip? y                          [DO YOU WANT THIS FILE (y/n)?]
  150 Opening BINARY mode data connection for snip.zip (17852 bytes).
  ###################################
  226 Transfer complete.
  ftp>
---------------------------------------

* You can also use 'wildcards' to ease the whole procedure of typing file-
names correctly:

 Example:
---------------------------------------
  ftp> mget ma*    [GET ALL FILES STARTING WITH 'ma' IN CURRENT DIRECTORY]
  mget maccell33.zip? y                     [DO YOU WANT THIS FILE (y/n)?]
  150 Opening BINARY mode data connection for maccel33.zip (9012 bytes).
  ##################
  226 Transfer complete.
  mget mauswind.zip? y                      [DO YOU WANT THIS FILE (y/n)?]
  150 Opening BINARY mode data connection for mauswind.zip (49373 bytes).
  ######################################################################
  ############################
  226 Transfer complete.
  mget maxidisk.zip.zip? y                  [DO YOU WANT THIS FILE (y/n)?]
  150 Opening BINARY mode data connection for maxidisk.zip (20970 bytes).
  ########################################
  226 Transfer complete.
  ftp>
---------------------------------------

* you can also use a 'wildcard' to get hold of single files or a specified
selection. Just be sure the wildcard is unique.

 i.e. if you want to download 'stzip26.lha' and there are no other files which
 start with the letters 'stz', you can type:  mget stz*.

 You can also download all files within a directory with a single
 command:  mget *

PS. you have to use 'mget' with wildcards- 'get' won't work!

Hint: if you're using 'mget' a lot and don't want to be asked if you want this
file or not, you can use the command 'prompt'. FTP will then proceed to get all
the files that match your wildcard without asking if you want them.

 Example:
---------------------------------------
  ftp> prompt                             [TURN OF 'GET THIS FILE (y/n)?']
  Interactive mode off.
  ftp>
---------------------------------------

[4] Ending your FTP session
---------------------------
close              [EXIT CURRENT FTP SITE, READY FOR A NEW FTP CONNECTION]

 Example:
---------------------------------------
  ftp> close       [EXIT CURRENT FTP SITE, READY FOR A NEW FTP CONNECTION]
  221 Goodbye.
  ftp>                      [FTP IS READY FOR A NEW CONNECTION- 'OPEN...']
---------------------------------------


quit                      [EXIT CURRENT FTP SITE/FTP MODE, RETURN TO UNIX]

 Example:
---------------------------------------
  ftp> quit               [EXIT CURRENT FTP SITE/FTP MODE, RETURN TO UNIX]
  oslonett.no%                                              [BACK IN UNIX]
---------------------------------------

* When doing a 'close' and re-entering a new FTP site you won't have to set
up features like 'bin', 'prompt' and 'hash' again.
You will however have to set it up again if you choose to 'quit', then
restart FTP.

A SHORT SUMMARY OF FTP COMMANDS/PROCEDURE
=========================================
ftp
open {FTP site address}                                    [OPEN FTP SITE]
Username: anonymous
Password: {your email address}

cd {directory/directory etc.}                            [GO TO DIRECTORY]
bin                                          [SET TO BINARY TRANSFER MODE]
hash      [DISPLAY '#####' CHARACTERS FOR VISUAL CONFIRMATION OF DOWNLOAD]
ls -l (or dir)                       [FILE/DIRECTORY LISTING WITH LENGTHS]
get {filename}                                         [GET A SINGLE FILE]
mget {filename filename filename filename}             [GET SEVERAL FILES]
quit                                         [EXIT FTP SITE/EXIT FTP MODE]


Other useful commands to know
=============================
stat (or status)                 [DISPLAYS USEFUL INFORMATION ABOUT SETUP]
prompt                               [TURNS ON/OFF 'GET THIS FILE (y/n)?']
get {filename} -  (i.e: get 00index -)         [DISPLAYS TEXT FILE ONLINE]
!host {FTP site}  (i.e: !host ftp.sn.no)        [SHOWS IP ADDRESS OF SITE]
help                                   [DISPLAYS A LIST OVER FTP COMMANDS]
help {command}    (i.e: help bin)                       [EXPLAINS COMMAND]

** Most of these commands work as 'on/off switches'. Each time you enter the
   command it switches over to the opposite position (from on to off or
   vice versa). If you're not sure about it's position, type:  stat
   (or: status) which will show your setup.

Be sure to check if transfer mode is set to 'bin'!!!! (if you try to
download binary files in ASCII mode all your downloads will be useless!-
You can download text files as binary as well, though you sometimes might
have to convert the end-of-line characters afterwards-
(get hold of 'unix2st.ttp' or similar program for this).

TRANSFERRING FILES TO MS-DOS DISKS ON UNIX TERMINAL SYSTEMS
-----------------------------------------------------------
Part of this is dependant on you having the GNU msdos-utilities available
at your site. If you don't, ask your local system operator how to transfer
files to/from MSDOS disks (The Atari ST reads MSDOS formatted disks).

* If your terminal doesn't have a disk drive you can "remote login" to
  another computer that does (and use it's disk drive):

rlogin {computer name}                   [REMOTE LOGIN TO ANOTHER MACHINE]
 (i.e:  rlogin hnoss)
(You can now access the disk just as if it was on your own terminal, except
for the disk drive being physically on that other machine).

mcopy {filename} a:                                 [COPY FILE(S) TO DISK]
 (i.e:  mcopy autosort.zip a:)
 (i.e:  mcopy autosort.zip icone.lzh files.txt dcxtrct.zoo a:)
mdir                                      [SHOW DISK DIRECTORY/FREE SPACE]
fdformat -l -d                                   [FORMAT 720K DISK- MSDOS]
fdformat -d                                    [FORMAT 1.44MB DISK- MSDOS]
mcopy a:{filename on disk} {copy filename}   [COPY FILE FROM DISK TO UNIX]
 (i.e:  mcopy a:ftplist.txt ftplist.txt)
mcopy a:"*" .                           [COPY ALL FILES FROM DISK TO UNIX]
mdel a:{filename}                                  [DELETE A FILE ON DISK]
 (i.e:  mdel a:ftplist.txt)
eject              [EJECT DISKS FROM DRIVES WITH NO 'MANUAL EJECT BUTTON']
